# Supermarkets API (Frontend Integration)

## Overview

- **Base URL**: your backend host (e.g. `http://127.0.0.1:8000`)
- **Auth**: Bearer JWT
  - Send header: `Authorization: Bearer <access_token>`
- **Roles**
  - **List + Detail**: `SUPERADMIN`, `PORTAL_USER`
  - **Create**: `SUPERADMIN`, `PORTAL_USER`
  - **Update**: `SUPERADMIN` only
  - **Delete (soft delete)**: `SUPERADMIN` only

## Standard response envelope

### Success

```json
{
  "data": {},
  "meta": null
}
```

### Error

```json
{
  "error": {
    "code": "SOME_CODE",
    "message": "Human readable message",
    "details": {},
    "request_id": "optional-request-id"
  }
}
```

Notes:
- `details` and `request_id` are optional.
- `code` is designed for frontend handling (switch/case).

## Common error codes

- **`UNAUTHENTICATED` (401)**: missing/invalid token
- **`UNAUTHORIZED` (403)**: token valid but role not allowed
- **`REQUEST_VALIDATION_ERROR` (400)**: request body/query validation failed (Pydantic)
- **`VALIDATION_ERROR` (400)**: backend validation (e.g., empty strings after trimming)
- **`CONFLICT` (409)**: uniqueness conflict (duplicate user_id/email/license/shop_id)
- **`RESOURCE_NOT_FOUND` (404)**: not found or soft-deleted

## GET `/supermarkets/` — List Supermarkets

**Auth**: `SUPERADMIN`, `PORTAL_USER`

### Query params

- `page` (int, default `1`, min `1`)
- `limit` (int, default `20`, min `1`, max `100`)

Optional search filters:
- `name` (string) — partial, case-insensitive match on shop name
- `user_id` (int) — exact match
- `shop_id` (string) — exact match
- `phone` (string) — exact match

### Response

- `data`: `SupermarketListItem[]`
- `meta`: pagination object

`SupermarketListItem`:
- `photo` (string | null)
- `shop_name` (string)
- `user_id` (int)
- `phone` (string | null)
- `location` (string) — street address
- `geo_coordinates` (object | null)
- `latitude` (number | null)
- `longitude` (number | null)

`meta`:
- `currentPage` (int)
- `limit` (int)
- `total` (int)
- `totalPages` (int)

### Example

Request:

```http
GET /supermarkets?page=1&limit=20&name=super&phone=9999999999
Authorization: Bearer <token>
```

Success response:

```json
{
  "data": [
    {
      "photo": "https://example.com/image.jpg",
      "shop_name": "Super Market 1",
      "user_id": 101,
      "phone": "9999999999",
      "location": "Main Street",
      "geo_coordinates": {"type":"Point","coordinates":[76.1,10.1]},
      "latitude": 10.1,
      "longitude": 76.1
    }
  ],
  "meta": { "currentPage": 1, "limit": 20, "total": 1, "totalPages": 1 }
}
```

## POST `/supermarkets/` — Create Supermarket

**Auth**: `SUPERADMIN`, `PORTAL_USER`

### Request body (`SupermarketCreateRequest`)

Required:
- `user_id` (int, > 0)
- `shop_name` (string, non-empty)
- `password` (string, non-empty)  
  - Stored **as-is** by backend (client should send hashed/encrypted as per your policy)
- `address` (object)
  - `street_address` (string, non-empty)
  - `city` (string, non-empty)
  - `state` (string, non-empty)
  - `pincode` (string, non-empty)
  - `latitude` (number | null)
  - `longitude` (number | null)

Optional shop fields:
- `phone` (string | null)
- `email` (string | null) — if provided must not be empty; must be unique
- `shop_license_no` (string | null) — if provided must not be empty; must be unique
- `photo` (string | null)
- `geo_coordinates` (object | null)
- `upi_id` (string | null)
- `delivery_time` (int | null, >= 0)

Optional nested:
- `subscription` (object | null)
  - `start_date` (string datetime | null)
  - `end_date` (string datetime | null)
  - `amount` (number/string decimal | null)
  - `status` (string enum, default `"active"`)
  - **Important**: if provided but incomplete, backend **ignores subscription** and creates supermarket without it
- `promotion` (object | null)
  - `promotion_link` (string | null)
  - `promotion_header` (string | null)
  - `promotion_content` (string | null)
  - `promotion_image_s3_key` (string | null)
  - `is_marketing_enabled` (bool, default `false`)

### Server-generated fields

- `shop_id` is generated by backend as: `SHOP{user_id}` (unique)

### Success response

Returns **the same object as** `GET /supermarkets/{user_id}` (see detail section), with:
- `meta: null`

### Errors

- `409 CONFLICT`: duplicate `user_id`, derived `shop_id`, `email`, or `shop_license_no`
- `400 REQUEST_VALIDATION_ERROR`: missing required fields / invalid types / invalid `delivery_time`
- `400 VALIDATION_ERROR`: empty strings for required fields after trimming

## GET `/supermarkets/{user_id}` — Get Supermarket Detail

**Auth**: `SUPERADMIN`, `PORTAL_USER`

### Path param

- `user_id` (int)

### Response (`SupermarketDetailResponse`)

Top-level:
- `shop_owner` (object)
- `address` (object)
- `subscription` (object | null)
- `promotion` (object | null)
- `delivery_partners` (array)
- `subscription_invoices` (array)
- `daily_order_stats` (array | null)

`shop_owner`:
- `shop_id` (string)
- `user_id` (int)
- `shop_name` (string)
- `phone` (string | null)
- `email` (string | null)
- `photo` (string | null)
- `status` (string)
- `payment_status` (string)
- `is_blocked` (bool)
- `geo_coordinates` (object | null)
- `upi_id` (string | null)
- `rating` (number | null)
- `delivery_time` (int | null)
- `created_at` (string datetime)
- `updated_at` (string datetime)

`address`:
- `street_address` (string)
- `city` (string)
- `state` (string)
- `pincode` (string)
- `latitude` (number | null)
- `longitude` (number | null)

`subscription` (if any):
- `subscription_id` (int)
- `start_date` (string datetime)
- `end_date` (string datetime)
- `amount` (number/string decimal)
- `status` (string)
- `last_payment_date` (string datetime | null)

`promotion` (if any):
- `promotion_link` (string | null)
- `promotion_header` (string | null)
- `promotion_content` (string | null)
- `promotion_image_s3_key` (string | null)
- `is_marketing_enabled` (bool)

`delivery_partners`:
- For **PORTAL_USER**: always `[]`
- For **SUPERADMIN**: array of:
  - `delivery_partner_id` (string)
  - `first_name` (string)
  - `last_name` (string | null)
  - `phone1` (int)
  - `email` (string | null)
  - `online_status` (string)
  - `current_status` (string)
  - `photo` (string)
  - `vehicle_detail` (string | null)
  - `rating` (number | null)
  - `created_at` (string datetime)

`subscription_invoices`:
- array of:
  - `invoice_id` (int)
  - `invoice_number` (string)
  - `billing_period_start` (string datetime)
  - `billing_period_end` (string datetime)
  - `amount` (number/string decimal)
  - `status` (string)
  - `document_type` (string)
  - `paid_at` (string datetime | null)
  - `created_at` (string datetime)

`daily_order_stats`:
- For **PORTAL_USER**: `null`
- For **SUPERADMIN**: array of:
  - `date` (string, e.g. `2026-03-25`)
  - `order_count` (int)
  - `total_amount` (number/string decimal)
  - `status_counts` (object `{ "<status>": <count> }`)

### Errors

- `404 RESOURCE_NOT_FOUND`: supermarket not found or soft-deleted

## PATCH `/supermarkets/{user_id}` — Update Supermarket (Partial)

**Auth**: `SUPERADMIN` only

### Path param

- `user_id` (int)

### Request body (`SupermarketUpdateRequest`) — all fields optional

ShopOwner fields:
- `shop_name` (string | null) — if provided must not be empty
- `phone` (string | null) — if provided must not be empty
- `email` (string | null) — if provided must not be empty; must be unique
- `shop_license_no` (string | null) — if provided must not be empty; must be unique
- `photo` (string | null)
- `geo_coordinates` (object | null)
- `upi_id` (string | null)
- `delivery_time` (int | null, >= 0)
- `status` (string enum | null)
- `payment_status` (string enum | null)
- `is_blocked` (bool | null)

Nested `address` (object | null):
- `street_address` (string | null) — if provided must not be empty
- `city` (string | null) — if provided must not be empty
- `state` (string | null) — if provided must not be empty
- `pincode` (string | null) — if provided must not be empty
- `latitude` (number | null)
- `longitude` (number | null)

### Response

Returns the updated detail object (same as `GET /supermarkets/{user_id}`), with:
- `meta: null`

### Errors

- `404 RESOURCE_NOT_FOUND`: supermarket not found / deleted
- `409 CONFLICT`: duplicate `email` or `shop_license_no`
- `400 REQUEST_VALIDATION_ERROR`: invalid types / invalid `delivery_time`
- `400 VALIDATION_ERROR`: empty strings (after trimming) for provided fields

## DELETE `/supermarkets/{user_id}` — Delete Supermarket (Soft Delete)

**Auth**: `SUPERADMIN` only

### Behavior

- Sets `shop_owners.is_deleted = true`
- Also sets `delivery_partners.is_deleted = true` for all partners under that supermarket’s `shop_id`

### Response

```json
{
  "data": { "deleted": true, "user_id": 123 },
  "meta": null
}
```

### Errors

- `404 RESOURCE_NOT_FOUND`: supermarket not found / already deleted
